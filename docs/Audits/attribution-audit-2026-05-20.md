## Attribution rendering logic (code baseline)

**`isKartalix` flag** (`worker-fetch-agent.js:6428–6430`):
```javascript
const isKartalix = !rawSource || rawSource === 'Kartalix' ||
  ['rewrite','original_synthesis','manual'].includes(a.publish_mode) ||
  (a.publish_mode && (a.publish_mode.startsWith('template') ||
   a.publish_mode.startsWith('youtube') || a.publish_mode === 'video_embed'));
```

**`srcUrl`** (`worker-fetch-agent.js:6438`):
```javascript
const srcUrl = a.url && a.url.startsWith('http') ? a.url : null;
```

**`attrHtml` logic** (`worker-fetch-agent.js:6471–6482`):
```javascript
if (!isKartalix && srcUrl)         → "Kaynak: <a href=srcUrl><strong>source</strong> →</a>"
if (mode==='rewrite' && srcUrl)    → "Kaynak temel alınarak ... üretildi: <a>source →</a>"
if (mode==='original_synthesis')   → "Birden fazla kaynaktan derlenerek ... üretildi."
else                               → "" (empty — no attribution rendered)
```

**`attrHtml` placement** (`worker-fetch-agent.js:6658`): AFTER `</div class="article-body">` and after any match-stats widget script. Always at the **bottom** of the article, inside `<article>` tag, before the reaction bar.

**Meta line** (`worker-fetch-agent.js:6585`):
```javascript
isKartalix ? 'Kartalix Editöryel · Ali Genç' : `📰 ${source}`
```

## Question 3 — Duhuliye-sourced articles: Duhuliye vs upstream source

**Answer: Attribution always shows Duhuliye, never the upstream source.**

The attribution logic uses `rawSource = a.source || a.source_name` and `srcUrl = a.url`. For articles fetched from Duhuliye:
- `source_name = 'Duhuliye'`
- `url = 'https://duhuliye.com/...'`

If the article mode is `rewrite`, attrHtml renders:
```
Kaynak temel alınarak … üretildi: <a href="https://duhuliye.com/..."><strong>Duhuliye</strong> →</a>
```

If Duhuliye's own article body says "Fanatik'in haberine göre…" (according to Fanatik), that upstream source is visible only in the body text (if Claude included it in the rewrite). There is no code that parses Duhuliye's source attribution or stores the upstream source separately. The reader who clicks "Duhuliye →" arrives at an aggregator page, not the primary source.

No `copy_source` or `upstream_source` field exists in the article shape.

## Root cause analysis

### Issue A — youtube_embed misattribution (highest severity)
`isKartalix` sets `true` for any mode starting with `'youtube'`, which forces the meta to show "Kartalix Editöryel · Ali Genç" and suppresses all attrHtml branches. This was likely intentional (Kartalix generates the intro paragraph + embed), but the effect is that A Spor and beIN Sports branded content appears as Kartalix-authored. The embedded iframe carries YouTube's own source label, but that's not a Kartalix attribution — it's YouTube's player UI.

**Fix shape:** In attrHtml, add a branch for `youtube_embed` that links to the YouTube watch URL regardless of isKartalix status. Or update `isKartalix` to not suppress the meta source for youtube_embed modes specifically — show "📰 A Spor (YouTube)" alongside the embed.

### Issue B — synthesis mode not covered by attrHtml
The attrHtml only handles `original_synthesis`. Plain `synthesis` mode (the more common Kartalix multi-source rewrite) falls through with no attribution. Both modes should show at least "Birden fazla kaynaktan derlenerek…"

**Fix shape:** Add `synthesis` to the `original_synthesis` branch in attrHtml, OR handle both with `['synthesis','original_synthesis'].includes(a.publish_mode)`.

### Issue C — attribution placement (below fold)
For AdSense compliance and journalistic standards, attribution should ideally appear near the byline (above the fold), not at the bottom after the body.

**Fix shape:** Move `${attrHtml}` from after `article-body` to inside `div.article-meta` (line 6584–6588), rendering as a third line in the meta block.

### Issue D — rabona_digest has no URL in KV
`src/publisher.js:2697` sets `url: ''` in the returned card, losing the `original_url: 'https://www.youtube.com/c/RabonaDigital'` that IS saved to Supabase. The KV article can't render a link because srcUrl is null.

**Fix shape:** Set `url: 'https://www.youtube.com/c/RabonaDigital'` in the returned card object, or add `original_url` handling to `srcUrl` derivation.

### Issue E — multi-source synthesis lists no sources
No `sources` array field in the article shape. Contributing source names are discarded at article generation time.

**Fix shape:** Store `sources: ['NTV Spor', 'Sabah', 'Hürriyet']` array in the KV article shape during synthesis, render a comma-separated "Kaynaklar: X, Y, Z" in attrHtml for synthesis/original_synthesis modes.

### Issue F — Duhuliye piercing
No upstream source tracking. Structural limitation — would require Duhuliye article scraping to identify the upstream source they cite.

**Fix shape (low priority):** Accept Duhuliye as the credited source. Add a UI label "Kaynak: Duhuliye (özet)" to signal it's an aggregator, not primary reporting.
