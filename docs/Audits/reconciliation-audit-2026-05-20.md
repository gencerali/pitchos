### 0.12 — Subtitle fix options

Option A (remove subtitle) is moot — no subtitle element exists in current code.

**Actual fix needed**: strip markdown heading markers from `bodyHtml`:
```javascript
// Current (worker-fetch-agent.js:6542):
bodyText.split('\n').map(l => l.trim() ? `<p>${escHtml(l)}</p>` : '').join('')

// Fix:
bodyText.split('\n').map(l => {
  const stripped = l.trim().replace(/^#+\s*/, '');
  return stripped ? `<p>${escHtml(stripped)}</p>` : '';
}).join('')
```
This prevents `# Title` from rendering as `# Title` in a paragraph.

### 4.1 — What exists now

**No `change_log` or `audit_log` table exists.** Migrations reviewed (0001–0013): none create a change_log table.

### 4.2 — Phase 1 scope (if approved)

**Proposed table** (no migration yet — this is a proposal):
```sql
CREATE TABLE change_log (
  id SERIAL PRIMARY KEY,
  ts TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  category TEXT NOT NULL,       -- 'admin_action', 'cron_failure', 'cost_threshold', 'config_change', 'deploy'
  scope TEXT,                   -- e.g. 'source:NTV Spor', 'site:BJK', 'article:slug'
  actor TEXT,                   -- 'admin', 'cron', 'system'
  summary TEXT NOT NULL,
  detail JSONB,
  related_to TEXT               -- slug, source name, etc.
);
```

**3-5 auto-capture points** (in priority order):
1. Manual admin promote (`worker-fetch-agent.js:1052`) — highest-value, admin-visible
2. Source config changes (enable/disable/bjk_filter toggle) — ops-critical
3. Cost cap threshold crossings (>50%, >80%, >100% of `MONTHLY_CLAUDE_CAP`)
4. Cron run failures / exception paths (where `processSite` throws)
5. Pool alarm events (already writing to KV; write to DB instead/also)

**Endpoints**:
- `POST /admin/log` — internal write endpoint (worker-to-worker, authenticated)
- `GET /admin/log?limit=20` — returns last N entries as JSON
- Panel on `/admin/report`: last 10 entries in a table

**Effort estimate**: ~200 lines (migration SQL + 2 endpoints + 5 instrumentation points + admin panel row). 2-3h work.
